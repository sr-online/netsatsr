# เว็บไซต์เช็กชื่อติว Netsat

โครงสร้างไฟล์:

- `index.html` — หน้าเช็กชื่อสำหรับนักเรียน (มือถือ/แท็บเล็ต) กรอกรหัสประจำตัวเพื่อลงชื่อ
- `dashboard.html` — หน้า Dashboard สำหรับครู (ต้องกรอกรหัสผ่านก่อนเข้าดู)
- `js/data.js` — ข้อมูลกิจกรรมและรายชื่อนักเรียน (คัดลอกจากไฟล์ PDF) แก้ไข/เพิ่มชื่อได้ที่ไฟล์นี้
- `js/firebase-init.js` — การเชื่อมต่อ Firebase
- `js/checkin.js` — ตรรกะหน้าเช็กชื่อ
- `js/dashboard.js` — ตรรกะหน้า Dashboard และการส่งออก Excel
- `firestore.rules` — กฎความปลอดภัยของฐานข้อมูล (นำไปวางใน Firebase Console)

## ใช้ Firebase โปรเจกต์เดียวกับระบบ "เว็บไซต์เช็กชื่อนักเรียนพิเศษ" แต่แยกข้อมูลกัน
ระบบนี้เก็บข้อมูลที่ collection **`netsat_checkins`** ส่วนอีกระบบเก็บที่ collection `checkins` จึงไม่ปนกัน ใช้ไฟล์ `firestore.rules` ในโปรเจกต์นี้แทนของเดิมได้เลย (มีกฎของทั้งสอง collection รวมอยู่ในไฟล์เดียว)

## ขั้นตอนติดตั้ง Firebase (ทำครั้งเดียว)

ไปที่ https://console.firebase.google.com/project/check-in-7989c

### 1. เปิดใช้งาน Firestore Database
ถ้ายังไม่เคยเปิด: Build → Firestore Database → Create database → เลือก production mode → เลือก location ที่ใกล้ (เช่น asia-southeast1) — ถ้าเปิดไว้แล้วจากระบบเดิม ข้ามขั้นตอนนี้ได้

### 2. ตั้งค่า Security Rules
Firestore Database → Rules → คัดลอกเนื้อหาจากไฟล์ `firestore.rules` ในโปรเจกต์นี้ไปวางแทนของเดิม แล้วกด Publish (ไฟล์นี้มีกฎครอบคลุมทั้ง collection `netsat_checkins` ของระบบนี้ และ `checkins` ของอีกระบบ)

### 3. อัปโหลดเว็บไซต์
อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์นี้ขึ้น hosting ที่รองรับไฟล์ static (เช่น Firebase Hosting, GitHub Pages) หรือทดสอบก่อนได้โดยดับเบิลคลิกเปิดไฟล์ `index.html` / `dashboard.html` โดยตรง (ต้องต่ออินเทอร์เน็ต เพราะโหลด Firebase SDK, ฟอนต์ และไลบรารี Excel จากอินเทอร์เน็ต)

## รหัสผ่าน Dashboard
ค่าเริ่มต้นคือ `netsat2569` แก้ไขได้ที่ตัวแปร `GATE_PASSWORD` ใน `js/dashboard.js` แนะนำให้เปลี่ยนก่อนใช้งานจริง

## การแก้ไขรายชื่อนักเรียน / ข้อมูลกิจกรรม
แก้ไขได้ที่ `js/data.js`
- `EVENT` = ชื่อกิจกรรมและสถานที่ (แสดงในหน้าเช็กชื่อ)
- `STUDENTS` = รายชื่อนักเรียนทั้งหมด key คือรหัสประจำตัว (string) value คือ `{ name, class }`

## การกันลงชื่อซ้ำ
แต่ละการลงชื่อจะถูกเก็บโดยใช้รหัสเอกสารที่คำนวณจาก รหัสประจำตัว + วันที่ (เช่น `15964_2569-07-14`) แทนการสุ่มรหัสเอกสาร ทำให้ถ้านักเรียนคนเดิมพยายามลงชื่อซ้ำในวันเดียวกัน Firestore Rules จะปฏิเสธการเขียนทับให้เองโดยอัตโนมัติ (เพราะกฎอนุญาตแค่ "สร้างใหม่" ไม่ใช่ "แก้ไขของเดิม") หน้าเว็บจะจับ error นี้แล้วแสดงข้อความ "ลงชื่อไปแล้ววันนี้" พร้อมเวลาที่ลงชื่อไว้ให้นักเรียนทราบ วิธีนี้ป้องกันได้แม้กดยืนยันซ้ำเร็วๆ หรือมีคนลงชื่อพร้อมกันจากหลายเครื่อง

## การเก็บข้อมูลแยกแต่ละวัน
ทุกรายการลงชื่อมีฟิลด์ `date` (เช่น `2026-07-14`) หน้า Dashboard มีตัวเลือกวันที่ให้เลือกดูข้อมูลย้อนหลังของแต่ละวันแยกกันได้

## Dashboard ครู
- เลือกวันที่ที่ต้องการดู (ค่าเริ่มต้นคือวันนี้)
- ดูสถิติ: จำนวนที่ลงชื่อแล้ว / จำนวนนักเรียนทั้งหมด / อัตราการเข้าร่วม
- สลับดูรายชื่อ "ลงชื่อแล้ว" หรือ "ยังไม่ลงชื่อ" พร้อมค้นหาด้วยชื่อหรือรหัสประจำตัว
- กด "ส่งออก Excel" เพื่อดาวน์โหลดไฟล์ .xlsx ของรายชื่อที่กำลังดูอยู่ (ใช้ไลบรารี SheetJS โหลดจาก CDN)

## หมายเหตุด้านความปลอดภัย
ระบบนี้เหมาะสำหรับใช้งานภายในโรงเรียน ข้อมูลที่เก็บมีเพียงรหัสประจำตัว/ชื่อ/ชั้น/เวลาที่ลงชื่อ ซึ่งไม่ใช่ข้อมูลอ่อนไหวมาก จึงตัดระบบล็อกอิน (Firebase Authentication) ออกเพื่อความง่ายในการใช้งานและติดตั้ง รหัสผ่านหน้า Dashboard เป็นการกันฝั่งหน้าเว็บแบบง่ายๆ เหมาะสำหรับกันนักเรียน/คนทั่วไปเดินผ่านๆ เข้ามาดูเท่านั้น
